Freshman Related Work

Cisco Cybersecurity Course

This year’s related topic was on cyber security, an extremely important and necessary part of the computer science industry. Without it, computers would not be safe to use and we wouldn’t even be close to where we are with technology today. Of what we learned about cyber security over the course of the school year, I believe that the cyber security cube was the most interesting topic that we went over.

The cyber security cube has three dimensions, and each dimension represents a fundamental part of the cyber security industry. Each dimension has three subcategories which make up the whole side of the cube. The first dimension of the cube is information states, which are the different states that data can be in. The subcategories that make it up are transmission, which is data being moved, storage, which is data at rest, and processing, which is data in use. The second dimension is countermeasures, which is about how cyber attacks are prevented from happening. The three subcategories are technology, policies and practices, and people. The last side is security principles, which is about how you can keep data and other information safe and available. The subcategories are confidentiality, integrity, and availability.

I thought that this was interesting to learn about because everything else that we learned about this year related back to the cyber security cube. Each side of the cube is very important, and if one of the sides went away, then cyber security would fail. If countermeasures were taken out of the cube, then we wouldn’t be able to defend against cyber attacks.

Sophomore Related Work

Cisco IFTTT Lab

Over the course of my sophomore year in computer science, we covered a wide range of subjects in the Cisco IOT fundamentals course. One of the main subjects that we covered was IFTTT (If This Then That). IFTTT is a public web service that enables creative control and automated interactions between various cloud based services like Twitter, Instagram, Facebook, Dropbox, Webex Teams, etc. The interactions are defined in IFTTT using small scripts called applets – as simple as “if” something happens, “then” trigger some action. This year I used IFTTT to send an email to me when it was going to rain.

The first thing that I did was create a program on IFTTT that utilized an applet called “weather underground” to add an event called sunrise and sunset to my google calendar. Then I had to build a circuit that connected with the red board, which was connected to an arduino. I ran multiple segments of python code that connected the arduino to the IFTTT program that I had made. Using the data collected from the photosensor, the arduino added sunlight and sunset events to my google calendar through IFTTT.

Before doing this assignment, I had never used the arduino’s to connect to something else like a website, so I didn’t really have an idea of what practical use they had. Using the Arduino and IFTTT to change my google calendar gave me an idea of what else they could be used for.

Junior Related Work

Related in my junior year was spent learning how to code and design games using Unity. The first game that I created was a version of Breakout, which was originally developed by Atari in 1976. The game is very simple, and a good place to start when learning to code in c#. While creating the game I learned how to create objects in the scene, how to put scripts on them, and many other useful things that are necessary to learn in order to code more complex games. This is also when I learned how to make things move using vectors and rigid bodies.

The second game that I created was called Sunnyland. It was a platformer, so it was much more complex than the brick breaker game, which only had two moving parts. In this game the player could move, jump, and climb walls. There were enemies that moved, and carefully designed levels, created using sprite tiles. I was able to use many of the skills that I had acquired in the previous project in this one, such as making things move. Allowing the player to wall-jump was a new challenge. I had to use a child object to detect of the player was close to a wall, and if they were, I wrote code to make it so they stuck and then slowly slid down.

Learning to code games in C# was actually quite easy, as I had alreadly learned the basics of coding in my sophomore year. This was when I realized that I would be able to apply my coding skills in any language. I wouldn't have to re-learn the basics every time, I would just have to learn the syntax of the new language and I would be able to code like normal.

Senior Related Work

Farming Game

While I did spend a little time in game design during related in my senior year, most of my time was spent working on my senior project. A senior project is a yearlong assignmnet that should dusplay all of the skills that you had learned throughout highschool. For my senior project I decided to create a farming game that would utilize my skills in Java, JavaFX, Scenebuilder, and SQL.

All of the items that are in my game are stored in a database created using MySQL and are retrieved when the game runs. There is one main items table and all the different types of items, such as seeds or tools, are stored in separate tables that reference the main table. The player moves on a grid pane created in scenebuilder. The bar at the bottom of the screen is a hotbar that can store items that the player puts in the slots. The player can select an item by clicking on the slot the item is in. The player can equip a hoe, hoe ground, and plant seeds in the ground. The in game time is calculated using a timer. The plant in the video can be harvested after growing for two days, but all plants can have different grow times and can drop different items. I worked hard on this project and believe that it accurately displays my programming skills.

I enjoyed creating this project because it allowed me to use all of the knowledge that I had gained from my junior and senior and show what I could do with it. If I continued to work on this game, I would really like to get local multiplayer working. I've attempted to make multiplayer games before, but I never really had the skills to do it. Now that I've completed four years of computer science, hopefully I can now accomplish this task.
